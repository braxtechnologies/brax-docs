# Contributing to [Project Name]

We welcome all contributions — from bug fixes and feature proposals to documentation and design improvements.

---

## 1. How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs, suggest features, or discuss improvements.
- Include clear steps to reproduce, version info, and screenshots if relevant.

### Submitting Pull Requests
1. Fork the repository and create a feature branch (`feature/my-change`).
2. Follow coding standards and ensure all tests pass.
3. Submit a pull request with a clear description of your change.
4. A maintainer will review and merge when ready.

---

## 2. Code Review Process
- All code changes must go through pull request review.
- At least one maintainer’s approval is required before merge.
- Larger or breaking changes may require Technical Steering Committee review.

---

## 3. Becoming a Maintainer
To become a Maintainer:
1. Contribute consistently over time (code, docs, reviews).
2. Demonstrate understanding of project direction and best practices.
3. Be nominated by an existing Maintainer or TSC member.
4. Be confirmed by the TSC (Type A) or via community vote (Type B).

Maintainers are expected to:
- Review contributions.
- Participate in discussions and releases.
- Enforce the Code of Conduct.

---

## 4. Contributor License
All contributors must agree to the **[CLA or DCO]** ensuring that contributions can be used under the project’s open-source license.
